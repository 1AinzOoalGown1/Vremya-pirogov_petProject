## Открытие и закрытие модального окна

- [ ] При клике на кнопку «Войти» (неавторизованный пользователь) открывается модальное окно авторизации
- [ ] Модальное окно содержит заголовок «Вход»
- [ ] Модальное окно закрывается при клике на крестик (×) в правом верхнем углу
- [ ] Модальное окно закрывается при клике на любое место вне модального окна (на оверлей)
- [ ] После закрытия окна состояние страницы не меняется (пользователь остаётся неавторизованным)

---

## Согласие на обработку персональных данных

- [ ] Отображается чекбокс с текстом: *«Я даю согласие на обработку моих персональных данных, в соответствии с Федеральным законом от 27.07.2006 г. №152-ФЗ "О персональных данных", на условиях, определенных политикой в области обработки и обеспечения безопасности персональных данных»*
- [ ] Текст «политикой» является кликабельной ссылкой на страницу с политикой обработки данных
- [ ] Чекбокс неактивен (не отмечен) по умолчанию
- [ ] Чекбокс можно отметить/снять кликом

### Валидация согласия
- [ ] При клике на любую кнопку авторизации («Войти через Telegram», «Войти через Max», «Войти через VK», «Войти через Яндекс», «Войти по номеру телефона») с **неотмеченным** чекбоксом под чекбоксом появляется сообщение: *«Чтобы продолжить, установите этот флажок.»*
- [ ] При клике на любую кнопку авторизации с **отмеченным** чекбоксом сообщение об ошибке **не** отображается
- [ ] При повторном клике на кнопку авторизации после установки флажка сообщение об ошибке исчезает

## Основное окно авторизации (способы входа)

- [ ] Отображаются кнопки: «Войти через Telegram», «Войти через Max», «Войти через VK», «Войти через Яндекс»
- [ ] У каждой кнопки отображается соответствующий логотип/иконка социальной сети/мессенджера
- [ ] Отображается текст «Другие способы авторизации» (кликабельный)
- [ ] При нажатии на «Другие способы авторизации» появляется раздел с надписью «ИЛИ» и кнопкой «Войти по номеру телефона»
- [ ] Кнопка «Войти по номеру телефона» отображается между надписью «ИЛИ» и кнопкой «Войти через Яндекс»

---

## Вход через Telegram

- [ ] При клике на «Войти через Telegram» с отмеченным чекбоксом открывается новое окно/вкладка с авторизацией через Telegram (бота)
- [ ] На странице сайта вместо модального окна с вариантами авторизации появляется окно ожидания
- [ ] В окне ожидания отображается крутящийся спиннер (кружок загрузки)
- [ ] В окне ожидания отображается текст: *«Пытаемся авторизовать Вас... Откройте мессенджер Telegram. В чате с ботом авторизации нажмите кнопку "Поделиться контактом"»*
- [ ] Кнопка «Поделиться контактом» открывает новое окно/вкладку с страницей авторизации через Telegram
- [ ] В окне ожидания отображается кнопка «Вернуться назад», при клике на которую возвращается модальное окно авторизации
- [ ] В окне ожидания отображается крестик (×), при клике на который окно ожидания закрывается (модальное окно авторизации не появляется)
- [ ] При успешной авторизации через Telegram модальное окно закрывается, кнопка «Войти» заменяется на иконку профиля
- [ ] При клике на иконку профиля выполняется переход на страницу `/profile`

## Вход по номеру телефона

### Открытие окна ввода номера
- [ ] При клике на «Войти по номеру телефона» (с отмеченным чекбоксом) отображается новое состояние модального окна
- [ ] В окне отображаются кнопки: «Войти через Telegram», «Войти через Max», «Войти через Яндекс» (VK отсутствует)
- [ ] Отображается текст: *«Подтвердите номер телефона. Чтобы войти или зарегистрироваться»*
- [ ] Отображается поле для ввода номера телефона

### Поле ввода номера телефона
- [ ] В поле можно вводить **только цифры** (буквы и спецсимволы не вводятся)
- [ ] Ограничение на ввод — **10 цифр** (не более)
- [ ] Маска ввода: `+7 (xxx) xxx-xx-xx` или аналогичный формат
- [ ] При вводе менее 10 цифр поле подсвечивается как незаполненное

### Кнопка «Получить код для входа»
- [ ] Отображается кнопка «Получить код для входа»
- [ ] При клике на кнопку с **пустым** полем номера появляется сообщение: *«Заполните это поле»*
- [ ] При клике на кнопку с **неполным** номером (менее 10 цифр) появляется сообщение: *«Заполните это поле»*
- [ ] При клике на кнопку с **корректным** номером (10 цифр) отображается новый экран с полем для ввода кода
- [ ] В новом экране отображается текст: *«На номер +7 (xxx) xxx-xx-xx отправлен код подтверждения. Введите его в поле ниже»*
- [ ] Номер телефона в тексте соответствует введённому ранее

### Поле ввода кода подтверждения
- [ ] Отображается поле из 4 квадратиков для ввода кода
- [ ] В поле можно вводить **только цифры** (буквы и спецсимволы не вводятся)
- [ ] Автоматический переход к следующему квадратику после ввода цифры
- [ ] При вводе 4 цифр автоматически активируется кнопка «Войти» (или она становится активной)
- [ ] Отображается кнопка «Вернуться назад»
- [ ] При клике на «Вернуться назад» возвращается предыдущее состояние окна (ввод номера телефона)
- [ ] Введённый номер телефона сохраняется при возврате

### Кнопка «Войти» (подтверждение кода)
- [ ] Отображается кнопка «Войти»
- [ ] Кнопка неактивна, пока не введены все 4 цифры кода
- [ ] При вводе **корректного** кода выполняется успешная авторизация
- [ ] При успешной авторизации модальное окно закрывается, кнопка «Войти» заменяется на иконку профиля
- [ ] При клике на иконку профиля выполняется переход на страницу `/profile`
- [ ] При вводе **некорректного** кода появляется сообщение: *«Не удалось авторизоваться. Код неверный»*
- [ ] Сообщение об ошибке отображается под полем ввода кода
- [ ] После отображения ошибки поле ввода кода очищается

---

## Успешная авторизация (общие проверки)

- [ ] После успешной авторизации любым способом модальное окно закрывается
- [ ] Кнопка «Войти» в хедере заменяется на иконку профиля (для авторизованного пользователя)
- [ ] При клике на иконку профиля выполняется переход на страницу `/profile`
- [ ] В профиле отображаются корректные данные авторизованного пользователя

---

## Обработка ошибок

- [ ] При любой ошибке авторизации пользователь остаётся на текущей странице (редирект не выполняется)
- [ ] Сообщения об ошибках отображаются в понятном для пользователя формате
- [ ] Пользователь может повторить попытку ввода после ошибки
